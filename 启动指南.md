# Personal Sous Chef 项目启动指南

本文档提供完整的项目启动步骤，包括数据库、后端和前端服务的启动方法。

## 📋 前置要求

### 必需软件
1. **Docker Desktop** - 用于运行 PostgreSQL 数据库
   - 下载地址：https://www.docker.com/products/docker-desktop
   - 确保 Docker Desktop 已启动并运行

2. **.NET SDK 10.0** - 用于运行 C# 后端
   - 下载地址：https://dotnet.microsoft.com/download
   - 验证安装：`dotnet --version`

3. **Flutter SDK** - 用于运行 Flutter 前端
   - 下载地址：https://flutter.dev/docs/get-started/install
   - 验证安装：`flutter --version`

4. **Android Studio** - 用于运行 Android 模拟器（或使用真机）

---

## 🚀 启动步骤

### 步骤 1: 启动数据库（PostgreSQL）

1. **确保 Docker Desktop 正在运行**
   - 检查 Docker 图标是否在菜单栏显示为运行状态
   - 验证命令：`docker ps`（应该能正常执行，不报错）

2. **在项目根目录启动数据库**
   ```bash
   cd /Users/chase/Documents/Internship/Projects/chef_git/A-team-PersonalSousChef
   docker-compose up -d postgres
   ```

3. **验证数据库是否启动成功**
   ```bash
   docker ps
   ```
   - 应该能看到 `souschef_postgres` 容器正在运行

4. **查看数据库日志（可选）**
   ```bash
   docker-compose logs postgres
   ```

---

### 步骤 2: 启动后端服务（C#）

1. **进入后端目录**
   ```bash
   cd backend-csharp
   ```

2. **首次运行：安装依赖**
   ```bash
   dotnet restore
   ```
   - 这会下载所有 NuGet 包（JWT、BCrypt、EF Core、PostgreSQL 等）

3. **启动后端服务**
   ```bash
   dotnet run
   ```

4. **验证后端是否启动成功**
   - 终端应该显示类似信息：
     ```
     Now listening on: http://0.0.0.0:5108
     Application started. Press Ctrl+C to shut down.
     ```
   - 后端服务运行在端口 **5108**

5. **测试后端 API（可选）**
   - 在浏览器中访问：`http://localhost:5108/hello`
   - 应该能看到 JSON 响应

**注意：** 保持此终端窗口打开，后端服务需要持续运行。

---

### 步骤 3: 启动前端应用（Flutter）

1. **打开新的终端窗口**（保持后端服务在第一个终端运行）

2. **进入前端目录**
   ```bash
   cd frontend-app
   ```

3. **首次运行：安装依赖**
   ```bash
   flutter pub get
   ```
   - 这会下载所有 Flutter 包（http、shared_preferences 等）

4. **检查可用设备**
   ```bash
   flutter devices
   ```
   - 确保能看到 Android 模拟器或连接的设备

5. **启动 Flutter 应用**
   ```bash
   flutter run
   ```
   - 或者指定设备：`flutter run -d <device-id>`

6. **等待应用编译和启动**
   - 首次运行可能需要几分钟
   - 应用启动后会自动打开在模拟器/设备上

## 📱 使用应用

### 首次使用流程

1. **启动页（Landing Page）**
   - 应用启动后显示登录/注册选项

2. **注册新用户**
   - 点击 "Sign up" 按钮
   - 填写：用户名、邮箱、密码、确认密码
   - 点击 "confirm" 完成注册
   - 注册成功后返回登录页面

3. **登录**
   - 点击 "Log in" 按钮
   - 输入：用户名或邮箱、密码
   - 点击 "Log in" 完成登录
   - 登录成功后进入主应用

4. **使用功能**
   - 查看用户信息：点击底部导航栏 "Me"
   - 编辑用户信息：在 Profile 页面点击 "Edit"
   - 管理偏好、禁忌、过敏：在 Profile 页面点击相应选项

## 🔧 常见问题排查

### 问题 1: Docker 无法启动数据库

**症状：** `docker-compose up -d postgres` 报错

**解决方案：**
1. 确保 Docker Desktop 已启动
2. 检查端口 5432 是否被占用：`lsof -i :5432`
3. 如果端口被占用，停止其他 PostgreSQL 服务或修改 `docker-compose.yml` 中的端口

### 问题 2: 后端无法连接数据库

**症状：** 后端启动时报错 "Connection refused" 或 "Unable to connect"

**解决方案：**
1. 确保数据库容器正在运行：`docker ps`
2. 检查数据库日志：`docker-compose logs postgres`
3. 验证连接字符串：检查 `backend-csharp/appsettings.json` 中的 `ConnectionStrings:DefaultConnection`
4. 等待数据库完全启动（可能需要 10-30 秒）

### 问题 3: 前端无法连接后端

**症状：** 登录/注册时显示 "Network error" 或 "Connection failed"

**解决方案：**
1. 确保后端服务正在运行（检查步骤 2）
2. 检查 API 配置：`frontend-app/lib/config/api_config.dart`
   - Android 模拟器使用：`10.0.2.2:5108`
   - 真机使用：你的电脑局域网 IP（如 `192.168.1.100:5108`）
3. 检查后端 CORS 配置是否正确
4. 在浏览器中测试后端 API：`http://localhost:5108/api/ums/auth/register`（应该返回错误，但能连接）

### 问题 4: Flutter 依赖安装失败

**症状：** `flutter pub get` 报错

**解决方案：**
1. 检查网络连接
2. 清理并重新获取：`flutter clean && flutter pub get`
3. 检查 `pubspec.yaml` 文件格式是否正确

### 问题 5: 数据库表未创建

**症状：** API 调用时返回数据库错误

**解决方案：**
1. 后端首次运行时会自动创建表（通过 EF Core `EnsureCreated()`）
2. 如果表未创建，停止后端服务，删除数据库容器，重新启动：
   ```bash
   docker-compose down
   docker volume rm a-team-personalsouschef_postgres_data
   docker-compose up -d postgres
   # 然后重新启动后端
   ```

## 🛑 停止服务

### 停止前端
- 在运行 Flutter 的终端按 `q` 或 `Ctrl+C`

### 停止后端
- 在运行后端的终端按 `Ctrl+C`

### 停止数据库
```bash
docker-compose down
```

### 停止并删除数据库数据（谨慎使用）
```bash
docker-compose down -v
```
⚠️ **警告：** 这会删除所有数据库数据！

## 📝 开发提示

### 热重载（Flutter）
- 在运行的应用中按 `r` 进行热重载
- 按 `R` 进行热重启
- 按 `q` 退出

### 查看后端日志
- 后端日志直接显示在运行 `dotnet run` 的终端中

### 查看数据库数据
- 使用数据库管理工具（如 pgAdmin、DBeaver）连接：
  - Host: `localhost`
  - Port: `5432`
  - Database: `souschef_db`
  - Username: `chef`
  - Password: `chef_password`

### 修改 API 配置
- 前端 API 地址：`frontend-app/lib/config/api_config.dart`
- 后端端口：`backend-csharp/Properties/launchSettings.json`
- 数据库连接：`backend-csharp/appsettings.json`
