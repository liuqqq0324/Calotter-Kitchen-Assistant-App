**更新时间**: 2025年12月16日 17:58  
**项目名称**: Calotter Personal Sous Chef Backend  
**版本**: 1.0.0-SNAPSHOT

---

## 1. 项目概述

Calotter 是一个基于 Spring Boot 3.2.0 和 JPA 的个人厨师助手后端系统，采用模块化设计，提供用户管理、库存管理、烹饪会话和健康营养追踪等功能。

### 1.1 核心特性

- **模块化架构**: 采用 Maven 多模块设计，实现业务模块解耦
- **JPA 数据持久化**: 使用 Spring Data JPA 进行数据库操作
- **统一响应格式**: 标准化的 API 响应结构
- **审计功能**: 自动记录创建和更新时间
- **JWT 认证**: 基于 JWT 的用户认证机制
- **事件驱动**: 使用 Spring 事件机制实现模块间解耦通信

## 4. 模块详细说明

### 4.1 calotter-common（通用模块）

**职责**: 提供所有业务模块共享的基础组件

**包结构**:
```
com.calotter.common/
├── config/
│   └── JpaAuditingConfig.java          # JPA 审计配置
├── core/
│   ├── domain/
│   │   ├── BaseEntity.java             # 实体基类（审计字段）
│   │   └── entity/
│   │       ├── RefAllergen.java        # 标准过敏原库
│   │       ├── StandardIngredient.java # 标准食材库
│   │       ├── StandardSpice.java      # 标准调料库
│   │       └── StandardUtensil.java    # 标准厨具库
│   ├── PageResult.java                 # 分页结果封装
│   └── Result.java                     # 统一响应结果
└── exception/
    └── GlobalExceptionHandler.java     # 全局异常处理器
```

**核心组件**:

1. **BaseEntity**: 所有实体的基类
   - `createTime`: 创建时间（自动填充）
   - `updateTime`: 更新时间（自动填充）
   - `createBy`: 创建人ID（自动填充）
   - `updateBy`: 更新人ID（自动填充）

2. **Result<T>**: 统一 API 响应格式
   - `code`: 响应状态码
   - `message`: 响应消息
   - `data`: 响应数据

3. **标准库实体**: 提供标准化的参考数据（食材、调料、厨具、过敏原）

## 5. 数据库设计

### 5.1 数据库配置

- **数据库类型**: PostgreSQL 15
- **连接池**: HikariCP（Spring Boot 默认）
- **DDL 策略**: `update`（开发环境自动更新表结构）
- **方言**: `PostgreSQLDialect`

### 5.2 主要数据表

#### 用户模块表
- `users` - 用户表
- `households` - 家庭组表
- `family_members` - 家庭成员表
- `health_goals` - 健康目标表

#### 库存模块表
- `household_ingredients` - 食材库存表
- `household_utensils` - 厨具表
- `household_spices` - 调料表
- `household_leftovers` - 剩菜表

#### 烹饪模块表
- `cooking_sessions` - 烹饪会话表
- `dishes` - 菜品表

#### 健康模块表
- `nutrition_logs` - 营养日志表
- `daily_nutrient_aggregates` - 每日营养聚合表

#### 标准库表
- `ref_standard_ingredients` - 标准食材库
- `ref_standard_utensils` - 标准厨具库
- `ref_standard_spices` - 标准调料库
- `ref_standard_allergens` - 标准过敏原库

### 5.3 数据特性

1. **JSONB 字段**: 使用 PostgreSQL JSONB 类型存储复杂数据结构
   - `User.settings`
   - `FamilyMember.preferences`
   - `CookingSession.requestContext`
   - `CookingSession.aiResponse`

2. **审计字段**: 所有实体继承 `BaseEntity`，自动记录：
   - `create_time`: 创建时间
   - `update_time`: 更新时间
   - `create_by`: 创建人ID
   - `update_by`: 更新人ID

3. **级联操作**: 
   - `Household` 删除时级联删除所有关联的库存数据
   - 使用 `orphanRemoval = true` 实现孤儿删除

## 6. API 设计

### 6.1 统一响应格式

所有 API 使用 `Result<T>` 作为响应格式：

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {}
}
```

### 6.2 主要 API 端点

#### 用户模块 (`/api/user`)
- `POST /api/user/register` - 用户注册
- `POST /api/user/login` - 用户登录
- `GET /api/user/profile` - 获取用户信息

#### 家庭模块 (`/api/household`)
- `POST /api/household` - 创建家庭组
- `GET /api/household/{id}` - 获取家庭信息
- `PUT /api/household/{id}` - 更新家庭信息
- `DELETE /api/household/{id}` - 删除家庭组

#### 库存模块 (`/api/inventory`)
- `GET /api/inventory/ingredients` - 获取食材列表
- `POST /api/inventory/ingredients` - 添加食材
- `PUT /api/inventory/ingredients/{id}` - 更新食材
- `DELETE /api/inventory/ingredients/{id}` - 删除食材
- （厨具、调料、剩菜类似）

#### 烹饪模块 (`/api/cooking`)
- `POST /api/cooking/generate-context` - 生成 AI 烹饪上下文
- `POST /api/cooking/start` - 开始烹饪会话
- `POST /api/cooking/complete` - 完成烹饪会话
- `GET /api/cooking/sessions` - 获取烹饪会话列表

#### 健康模块 (`/api/health`)
- `POST /api/health/nutrition/log` - 记录营养日志
- `GET /api/health/nutrition/daily` - 获取每日营养数据
- `GET /api/health/nutrition/weekly` - 获取周报

## 7. 安全机制

### 7.1 认证方式

- **JWT Token**: 使用 JWT 进行用户认证
- **密码加密**: 使用 Spring Security 的 BCryptPasswordEncoder

### 7.2 JWT 配置

- **密钥**: 配置在 `application.yml` 中
- **发行者**: CalotterBackend
- **受众**: CalotterFrontend
- **过期时间**: 3000 秒（50 分钟）

### 7.3 安全配置

- Spring Security 配置在 `calotter-user` 模块的 `SecurityConfig` 中
- 当前为开发阶段，部分接口可能未完全启用安全验证

## 8. 事件驱动机制

### 8.1 事件类型

1. **CookingSessionCompletedEvent** (烹饪模块发布)
   - 触发时机: 烹饪会话完成时
   - 监听者: `CookingSessionCompletedEventListener` (健康模块)

2. **NutritionLogCreatedEvent** (健康模块发布)
   - 触发时机: 营养日志创建时
   - 监听者: `NutritionLogEventListener` (健康模块内部)

### 8.2 异步处理

- 健康模块使用 `@Async` 注解实现异步处理
- 配置在 `AsyncConfig` 中

## 9. 开发与部署

### 9.1 开发环境要求

- Java 17+
- Maven 3.6+
- PostgreSQL 15+
- IDE（推荐 IntelliJ IDEA 或 Eclipse）

### 9.2 本地开发流程

1. **启动数据库**
   ```bash
   docker-compose up -d
   ```

2. **配置数据库连接**
   编辑 `calotter-start/src/main/resources/application.yml`

3. **编译项目**
   ```bash
   mvn clean install
   ```

4. **启动应用**
   ```bash
   cd calotter-start
   mvn spring-boot:run
   ```
   或直接运行 `CalotterApplication.java`

### 9.3 Docker 支持

项目包含 `docker-compose.yml`，可快速启动 PostgreSQL 数据库：

```yaml
services:
  postgres:
    image: postgres:15-alpine
    ports:
      - "5432:5432"
    environment:
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: "123"
      POSTGRES_DB: calotter
```

### 9.4 测试

各模块包含单元测试：
- `calotter-user`: 用户服务测试、家庭服务测试
- `calotter-cooking`: 烹饪会话服务测试、菜品服务测试、剩菜服务测试
- `calotter-inventory`: 库存服务测试
- `calotter-health`: 营养日志服务测试、营养聚合服务测试

运行测试：

## 10. 架构特点与优势

### 10.1 模块化设计

- **高内聚低耦合**: 每个模块职责明确，模块间依赖清晰
- **易于扩展**: 新增功能只需添加新模块或扩展现有模块
- **独立测试**: 各模块可独立进行单元测试

### 10.2 统一的基础设施

- **BaseEntity**: 统一的审计字段管理
- **Result**: 统一的 API 响应格式
- **GlobalExceptionHandler**: 统一的异常处理
- **标准库**: 统一的数据参考标准

### 10.3 事件驱动

- 模块间通过事件进行解耦通信
- 支持异步处理，提高系统响应性能

### 10.4 数据持久化

- 使用 JPA 简化数据库操作
- 支持 PostgreSQL JSONB 类型，灵活存储复杂数据
- 自动审计字段填充，便于数据追踪
