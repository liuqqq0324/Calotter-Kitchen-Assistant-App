## 详细步骤

### 步骤 1: 停止现有服务

```bash
# 停止所有运行中的 Java 后端服务
pkill -f "calotter" || true

# 停止 Docker 容器
cd /Users/chase/Documents/Internship/Projects/chef_git/A-team-PersonalSousChef/backend-calotter
docker-compose down
```

### 步骤 2: 清理数据库

**⚠️ 警告：此操作将删除所有数据库数据！**

```bash
# 方式 1: 删除 Docker Volume（完全清理，推荐）
docker-compose down -v

# 方式 2: 仅删除数据库内容（保留容器配置）
docker-compose down
docker volume rm backend-calotter_postgres_data 2>/dev/null || true
```

### 步骤 3: 启动 PostgreSQL 容器

```bash
# 启动 PostgreSQL
docker-compose up -d

# 等待数据库就绪（约 5-10 秒）
echo "等待 PostgreSQL 启动..."
sleep 8

# 验证容器状态
docker ps | grep calotter_postgres

# 验证数据库连接
docker exec calotter_postgres pg_isready -U postgres
```

### 步骤 4: 重建数据库

项目使用 **JPA 的 `ddl-auto: update`** 自动创建表结构，因此：

1. **删除现有数据库**（如果存在）
   ```bash
   docker exec calotter_postgres psql -U postgres -c "DROP DATABASE IF EXISTS calotter;"
   ```

2. **创建新数据库**
   ```bash
   docker exec calotter_postgres psql -U postgres -c "CREATE DATABASE calotter;"
   ```

3. **验证数据库创建**
   ```bash
   docker exec calotter_postgres psql -U postgres -c "\l" | grep calotter
   ```

### 步骤 5: 编译项目

```bash
# 在项目根目录
cd /Users/chase/Documents/Internship/Projects/chef_git/A-team-PersonalSousChef/backend-calotter

# 清理并编译
mvn clean install -DskipTests
```

### 步骤 6: 启动后端服务

```bash
# 进入启动模块目录
cd calotter-start

# 启动应用（前台运行，查看日志）
mvn spring-boot:run

# 或者后台运行
nohup mvn spring-boot:run > ../logs/backend.log 2>&1 &
echo "后端服务 PID: $!"
```

**服务地址**: `http://localhost:8080`

### 步骤 7: 验证服务

```bash
# 检查服务是否启动
curl http://localhost:8080/actuator/health 2>/dev/null || echo "服务可能还在启动中..."

# 查看日志（如果后台运行）
tail -f ../logs/backend.log
```

## 常见问题

### 问题 1: 端口 5432 被占用

```bash
# 查找占用端口的进程
lsof -i :5432

# 停止占用端口的进程
kill -9 <PID>
```

### 问题 2: 数据库连接失败

**检查项**：
1. 确认容器正在运行
   ```bash
   docker ps | grep calotter_postgres
   ```

2. 检查容器日志
   ```bash
   docker logs calotter_postgres
   ```

3. 测试数据库连接
   ```bash
   docker exec calotter_postgres psql -U postgres -c "SELECT version();"
   ```

### 问题 3: JPA 表创建失败

**可能原因**：
- 数据库连接配置错误
- 实体类定义有问题

**解决方法**：
1. 检查 `application.yml` 中的数据库配置
2. 查看启动日志中的错误信息
3. 确认所有实体类正确继承 `BaseEntity`

### 问题 4: Maven 编译失败

```bash
# 清理并重新编译
mvn clean install -U

# 如果仍有问题，检查 Java 版本
java -version  # 应该是 17+
```

### 问题 5: 后端服务无法启动

**检查项**：
1. 确认数据库已启动并可以连接
2. 检查端口 8080 是否被占用
   ```bash
   lsof -i :8080
   ```
3. 查看详细日志
   ```bash
   tail -f logs/backend.log
   ```

## 开发环境快速命令

### 查看数据库表

```bash
# 连接数据库
docker exec -it calotter_postgres psql -U postgres -d calotter

# 在 psql 中执行
\dt  # 列出所有表
\dn  # 列出所有 schema
SELECT table_name FROM information_schema.tables WHERE table_schema = 'public';
```

### 查看容器日志

```bash
# PostgreSQL 日志
docker logs -f calotter_postgres

# 后端服务日志（如果后台运行）
tail -f logs/backend.log
```

### 停止所有服务

```bash
# 停止后端服务
pkill -f "calotter" || true

# 停止数据库容器
docker-compose down
```

### 完全清理（包括数据卷）

```bash
# 停止并删除容器和数据卷
docker-compose down -v

# 清理 Maven 构建文件
mvn clean
```

## 注意事项

1. **数据备份**: 执行数据库重建前，请确保已备份重要数据
2. **JPA 自动创建**: 项目使用 `ddl-auto: update`，首次启动会自动创建表结构
3. **开发环境**: 此配置适用于开发环境，生产环境建议使用 Flyway 或 Liquibase
4. **端口冲突**: 确保 5432（PostgreSQL）和 8080（后端服务）端口未被占用
