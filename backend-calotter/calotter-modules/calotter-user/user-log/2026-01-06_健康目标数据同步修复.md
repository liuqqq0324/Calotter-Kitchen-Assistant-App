## 📊 功能现状概述

### 功能简介
健康目标（Health Goal）功能允许用户：
1. **查看BMI**: 根据用户的身高、体重自动计算并显示BMI指数
2. **选择健康目标**: 支持三种目标类型
   - `MAINTENANCE` - 保持健康（BMI指数非常健康）
   - `LOSE_FAT` - 减脂
   - `MUSCLE_GAIN` - 增肌
3. **获取营养建议**: 基于用户身体信息和健康目标，通过AI生成个性化的每日营养目标
   - 每日卡路里（Daily Calories）
   - 蛋白质（Protein，单位：g）
   - 脂肪（Fat，单位：g）
   - 碳水化合物（Carbohydrates，单位：g）
   - 纤维（Fiber，单位：g）

### 技术架构

#### 数据库实体
- **HealthGoal** (`health_goals` 表)
  - 关联 `User` 实体（ManyToOne）
  - 存储目标类型、活动水平、初始身体数据快照
  - 存储AI生成的营养目标数据
  - 支持状态管理（ACTIVE=1, ARCHIVED=0）

- **User** (`users` 表)
  - `currentHeight` (Integer) - 当前身高（cm）
  - `currentWeight` (BigDecimal) - 当前体重（kg）
  - `gender` (Integer) - 性别（1=Male, 0=Female）
  - `birthdate` (LocalDate) - 出生日期（用于计算年龄）
  - `settings` (Map<String, Object>) - 用户设置，包含 `profile` 子对象

#### API端点
1. **POST `/api/user/health-goal`** - 创建/更新健康目标
   - 请求体: `{ "goalType": "LOSE_FAT", "activityLevel": 1.55 }`（activityLevel可选）
   - 响应: `HealthGoalResponse`（包含营养目标数据）

2. **GET `/api/user/health-info`** - 获取用户健康信息
   - 响应: `UserHealthInfoResponse`（包含BMI、目标类型、营养目标）

#### AI服务
- **Groq AI** (`GroqAiNutritionRecommendationService`)
  - 模型: `llama-3.3-70b-versatile`
  - 输入: 身高、体重、年龄、性别、BMI、目标类型、活动水平
  - 输出: JSON格式的营养建议（卡路里、蛋白质、脂肪、碳水、纤维）
  - 配置: `application.yml` 中的 `ai.api.groq.*` 配置项

### 3. 数据存储位置说明

#### 用户身体数据的两处存储
1. **User 实体直接字段**（推荐）
   - `User.currentHeight` (Integer)
   - `User.currentWeight` (BigDecimal)
   - 用途: 后端验证、BMI计算、AI服务输入

2. **User.settings.profile**（兼容旧数据）
   - `User.settings.get("profile").get("height")` (Object)
   - `User.settings.get("profile").get("weight")` (Object)
   - 用途: 前端显示、兼容旧版本数据

#### 数据同步机制（本次修复的核心）
- **问题**: 前端可能只更新了 `settings.profile`，但 `User.currentHeight/currentWeight` 为 null
- **解决方案**: 在创建健康目标时，自动从 `settings.profile` 同步到实体字段
- **时机**: `UserHealthService.createOrUpdateHealthGoal()` 的验证阶段

## 📋 问题描述

### 问题1：数据不一致导致验证失败
- **现象**: 前端可以显示用户的身高和体重（从 `User.settings.profile` 读取），但保存健康目标时后端返回 400 错误："用户身高或体重信息不完整"
- **根本原因**: 
  - 前端显示数据来自 `User.settings.profile.height` 和 `User.settings.profile.weight`
  - 后端验证检查的是 `User.currentHeight` 和 `User.currentWeight` 直接字段
  - 当直接字段为 null 时，即使 `settings.profile` 中有数据，验证也会失败

### 问题2：调试信息不足
- 缺少数据同步过程的详细日志
- 无法追踪数据从 `settings.profile` 到实体字段的转换过程
- AI 服务调用和保存过程的日志不够详细

### 1. UserHealthService.java - 数据同步逻辑

#### 修改位置
`createOrUpdateHealthGoal()` 方法中的用户身体信息验证部分

#### 修改前
```java
// 2. 验证用户身体信息
if (user.getCurrentHeight() == null || user.getCurrentWeight() == null) {
    throw new IllegalArgumentException("用户身高或体重信息不完整，请先完善个人信息");

```

#### 修改后
```java
// 2. 验证用户身体信息
log.info("🔍 验证用户身体信息:");
log.info("   User.currentHeight: {}", user.getCurrentHeight());
log.info("   User.currentWeight: {}", user.getCurrentWeight());

if (user.getCurrentHeight() == null || user.getCurrentWeight() == null) {
    log.warn("   ⚠️ User 实体字段为空，尝试从 settings.profile 同步数据");
    
    // 尝试从 settings.profile 中获取并同步
    Map<String, Object> settings = user.getSettings();
    if (settings != null) {
        log.info("   User.settings 存在，查找 profile 数据");
        @SuppressWarnings("unchecked")
        Map<String, Object> profile = (Map<String, Object>) settings.get("profile");
        if (profile != null) {
            log.info("   User.settings.profile 存在，查找 height 和 weight");
            Object heightObj = profile.get("height");
            Object weightObj = profile.get("weight");
            log.info("   从 settings.profile 获取: height={} (type: {}), weight={} (type: {})", 
                    heightObj, heightObj != null ? heightObj.getClass().getSimpleName() : "null",
                    weightObj, weightObj != null ? weightObj.getClass().getSimpleName() : "null");
            
            if (heightObj != null && weightObj != null) {
                // 同步到 User 实体字段
                try {
                    Integer height = Integer.parseInt(heightObj.toString());
                    BigDecimal weight = new BigDecimal(weightObj.toString());
                    user.setCurrentHeight(height);
                    user.setCurrentWeight(weight);
                    userRepository.save(user);
                    log.info("   ✅ 已同步 profile 数据到 User 实体字段: height={}cm, weight={}kg", height, weight);
                } catch (Exception e) {
                    log.error("   ❌ 同步 profile 数据失败: {}", e.getMessage(), e);
                }
            } else {
                log.warn("   ⚠️ settings.profile 中 height 或 weight 为 null");
            }
        } else {
            log.warn("   ⚠️ User.settings.profile 不存在");
        }
    } else {
        log.warn("   ⚠️ User.settings 为 null");
    }
    
    // 再次检查
    if (user.getCurrentHeight() == null || user.getCurrentWeight() == null) {
        log.error("   ❌ 用户身高或体重信息不完整，无法创建健康目标");
        log.error("   最终状态: currentHeight={}, currentWeight={}", 
                user.getCurrentHeight(), user.getCurrentWeight());
        throw new IllegalArgumentException("用户身高或体重信息不完整，请先完善个人信息");
    }
}

log.info("   ✅ 用户身体信息验证通过: height={}cm, weight={}kg", 
        user.getCurrentHeight(), user.getCurrentWeight());
```

#### 关键改进
1. **自动数据同步**: 当 `User.currentHeight` 或 `User.currentWeight` 为 null 时，自动从 `User.settings.profile` 读取并同步
2. **数据持久化**: 同步后立即保存到数据库，确保数据一致性
3. **类型转换**: 处理 `height`（Integer）和 `weight`（BigDecimal）的类型转换
4. **异常处理**: 同步失败时记录详细错误信息，但不中断流程（会再次验证）
