# 厨房邀请功能使用说明

**创建日期**: 2026-01-12  
**功能类型**: 新功能实现  
**影响范围**: calotter-user 模块（HouseholdService, HouseholdController）+ 前端 Flutter 应用

---

## 📊 功能概述

### 功能简介
厨房邀请功能允许用户：
1. **创建厨房**：用户注册时自动创建默认厨房，也可以手动创建新厨房
2. **邀请用户**：通过用户名或邮箱邀请已注册用户加入厨房
3. **通过邀请码加入**：使用6位字母数字邀请码加入厨房
4. **二维码邀请**：生成二维码，扫描后自动加入厨房
5. **管理厨房**：查看已加入的厨房列表、切换当前厨房、退出厨房
6. **重新生成邀请码**：厨房所有者可以重新生成邀请码（旧邀请码失效）

### 技术架构

#### 数据库实体
- **Household** (`households` 表)
  - `id`: 主键
  - `name`: 厨房名称
  - `inviteCode`: 6位字母数字邀请码（唯一）
  - `ownerId`: 所有者用户ID
  - `members`: 多对多关联到 `User` 实体（通过 `users_households` 中间表）

- **User** (`users` 表)
  - `id`: 主键
  - `joinedHouseholds`: 多对多关联到 `Household` 实体（用户加入的所有厨房）
  - `currentHouseholdId`: 当前活跃的厨房ID

#### 后端服务
- **HouseholdService**: 核心业务逻辑
  - `createHousehold()`: 创建厨房并生成邀请码
  - `inviteUserToHousehold()`: 通过用户名/邮箱邀请用户
  - `joinHouseholdByInviteCode()`: 通过邀请码加入厨房
  - `leaveHousehold()`: 退出厨房
  - `removeMemberFromHousehold()`: 所有者移除成员
  - `switchCurrentHousehold()`: 切换当前厨房
  - `getJoinedHouseholds()`: 获取用户加入的所有厨房
  - `regenerateInviteCode()`: 重新生成邀请码

- **HouseholdController**: REST API 端点
  - `POST /api/household`: 创建厨房
  - `POST /api/household/{householdId}/invite`: 邀请用户
  - `POST /api/household/join`: 通过邀请码加入
  - `DELETE /api/household/{householdId}/leave`: 退出厨房
  - `DELETE /api/household/{householdId}/members/{memberId}`: 移除成员
  - `PUT /api/household/{householdId}/switch`: 切换当前厨房
  - `GET /api/household/user/{userId}/joined`: 获取已加入的厨房列表
  - `PUT /api/household/{householdId}/regenerate-invite-code`: 重新生成邀请码

#### 前端实现
- **HouseholdManagePage**: 厨房管理页面
  - 显示当前厨房信息（邀请码、二维码）
  - 扫描二维码加入
  - 输入邀请码加入
  - 查看已加入的厨房列表
  - 切换/退出厨房
  - 邀请用户（仅 owner）

- **ScanJoinHouseholdPage**: 二维码扫描页面
  - 使用 `mobile_scanner` 扫描二维码
  - 自动解析邀请码并加入厨房

- **InviteQrCodeWidget**: 二维码生成组件
  - 使用 `qr_flutter` 生成二维码
  - 显示邀请码和厨房名称

## 🔄 数据流说明

### 1. 用户注册流程

```
用户注册
  ↓
UserService.register()
  ↓
创建 User 实体
  ↓
HouseholdService.createHousehold()
  ↓
创建 Household 实体
  ├─ 生成6位邀请码（唯一）
  ├─ 设置 ownerId = userId
  └─ 保存到数据库
  ↓
User.setCurrentHouseholdId(householdId)
User.getJoinedHouseholds().add(household)
  ↓
保存 User 实体
  ↓
返回 AuthResponse（包含 householdId）
```

**数据库变化**:
- `users` 表：新增用户记录
- `households` 表：新增厨房记录（inviteCode 自动生成）
- `users_households` 表：新增关联记录（user_id, household_id）

### 2. 通过邀请码加入流程

```
用户B输入邀请码
  ↓
前端调用 HouseholdService.joinHousehold(inviteCode)
  ↓
POST /api/household/join?inviteCode=ABC123&userId=2
  ↓
HouseholdService.joinHouseholdByInviteCode()
  ├─ 通过 inviteCode 查找 Household
  ├─ 验证用户是否已加入
  ├─ 添加关联：User.getJoinedHouseholds().add(household)
  └─ 自动切换：User.setCurrentHouseholdId(householdId)
  ↓
保存 User 实体
  ↓
返回 HouseholdResponse
  ↓
前端更新本地存储（householdId）
前端刷新厨房列表
```

**数据库变化**:
- `users_households` 表：新增关联记录（user_id=2, household_id=1）
- `users` 表：更新 `currentHouseholdId = 1`

### 3. 通过用户名/邮箱邀请流程

```
Owner 输入用户名/邮箱
  ↓
前端调用 HouseholdService.inviteUser()
  ↓
POST /api/household/{householdId}/invite?usernameOrEmail=userB&inviterId=1
  ↓
HouseholdService.inviteUserToHousehold()
  ├─ 验证邀请人权限（owner 或 member）
  ├─ 通过 usernameOrEmail 查找被邀请用户
  ├─ 验证用户是否已加入
  └─ 添加关联：invitedUser.getJoinedHouseholds().add(household)
  ↓
保存 User 实体（被邀请用户）
  ↓
返回成功响应
```

**数据库变化**:
- `users_households` 表：新增关联记录（user_id=被邀请用户ID, household_id=厨房ID）

**注意**: 被邀请用户不会自动切换当前厨房，需要手动切换

### 4. 切换当前厨房流程

```
用户点击 "Switch" 按钮
  ↓
前端调用 HouseholdService.switchCurrentHousehold()
  ↓
PUT /api/household/{householdId}/switch?userId=2
  ↓
HouseholdService.switchCurrentHousehold()
  ├─ 验证用户已加入该厨房
  └─ 更新：User.setCurrentHouseholdId(newHouseholdId)
  ↓
保存 User 实体
  ↓
前端更新本地存储（householdId）
前端刷新页面
```

**数据库变化**:
- `users` 表：更新 `currentHouseholdId` 字段

### 5. 退出厨房流程

```
用户点击 "Leave" 按钮
  ↓
前端显示确认对话框
  ↓
用户确认退出
  ↓
前端调用 HouseholdService.leaveHousehold()
  ↓
DELETE /api/household/{householdId}/leave?userId=2
  ↓
HouseholdService.leaveHousehold()
  ├─ 验证用户不是 owner（owner 不能退出）
  ├─ 验证用户已加入该厨房
  ├─ 移除关联：User.getJoinedHouseholds().remove(household)
  └─ 如果退出的是当前厨房，清空 currentHouseholdId
  ↓
保存 User 实体
  ↓
前端清空本地存储（如果退出的是当前厨房）
前端刷新列表
```

**数据库变化**:
- `users_households` 表：删除关联记录
- `users` 表：如果退出的是当前厨房，`currentHouseholdId` 设为 null

### 6. 重新生成邀请码流程

```
Owner 点击 "Regenerate Invite Code"
  ↓
前端调用 HouseholdService.regenerateInviteCode()
  ↓
PUT /api/household/{householdId}/regenerate-invite-code?ownerId=1
  ↓
HouseholdService.regenerateInviteCode()
  ├─ 验证用户是 owner
  ├─ 生成新的6位邀请码（确保唯一）
  └─ 更新：Household.setInviteCode(newInviteCode)
  ↓
保存 Household 实体
  ↓
返回新的 HouseholdResponse
  ↓
前端更新显示的邀请码和二维码
```

**数据库变化**:
- `households` 表：更新 `inviteCode` 字段
- **旧邀请码失效**：使用旧邀请码无法加入

## ⚠️ 注意事项

### 1. 邀请码唯一性
- 邀请码是6位字母数字组合（例如：`ABC123`）
- 系统自动确保唯一性
- 重新生成后，旧邀请码立即失效

### 2. 当前厨房逻辑
- 用户注册时自动创建默认厨房并设为当前厨房
- 通过邀请码加入时，如果用户没有当前厨房，自动设为新加入的厨房
- 如果用户已有当前厨房，加入新厨房时也会自动切换（方便使用）
- 退出当前厨房后，`currentHouseholdId` 会被清空

### 3. Owner 权限限制
- Owner 不能退出自己的厨房
- 只有 Owner 可以重新生成邀请码
- 只有 Owner 可以移除成员（未来功能）

### 4. 数据共享范围
- **共享数据**（在厨房级别）:
  - 库存（Ingredient, HouseholdUtensil, HouseholdSpice）
  - 剩菜（LeftoverDish）
  - 烹饪会话（CookingSession）
  - 菜谱（Dish）

- **个人数据**（在用户级别）:
  - 健康目标（HealthGoal）
  - 营养日志（NutritionLog）
  - 每日营养汇总（DailyNutrientAggregate）
  - 用户偏好（Preferences, Allergies）

### 5. 多厨房支持
- 用户可以同时属于多个厨房
- 但只能有一个当前活跃的厨房（`currentHouseholdId`）
- 所有厨房操作（库存、烹饪等）都基于当前厨房

## 🐛 常见问题

### Q1: 邀请码无效怎么办？
**A**: 
- 检查邀请码是否正确输入（6位字母数字）
- 确认邀请码未被重新生成（旧邀请码会失效）
- 确认厨房未被删除

### Q2: 为什么看不到 "Leave" 按钮？
**A**: 
- 如果你是厨房 Owner，不能退出自己的厨房
- 只能删除整个厨房（未来功能）

### Q3: 扫描二维码没反应？
**A**: 
- 确认已授予相机权限
- 检查二维码是否完整显示
- 确认二维码包含有效的邀请码

### Q4: 加入厨房后数据不显示？
**A**: 
- 确认已切换到新加入的厨房（点击 "Switch"）
- 刷新页面（下拉刷新）
- 检查网络连接

### Q5: 如何删除厨房？
**A**: 
- 目前只能通过后端 API 删除（Owner 权限）
- 前端删除功能待实现
