### 2. 语音助手服务修改

#### ✅ **cooking_voice_assistant.dart** - 修改
- **位置**: `frontend-app/lib/services/cooking_voice_assistant.dart`
- **修改内容**:

1. **添加导入**:
   ```dart
   import 'package:permission_handler/permission_handler.dart';
   ```

2. **新增方法 `checkAndRequestPermission()`**:
   ```dart
   /// 检查并请求麦克风权限
   Future<bool> checkAndRequestPermission() async {
     // 检查权限状态
     final status = await Permission.microphone.status;
     
     if (status.isGranted) {
       return true;
     }
     
     if (status.isDenied) {
       // 请求权限
       final result = await Permission.microphone.request();
       return result.isGranted;
     }
     
     if (status.isPermanentlyDenied) {
       return false;  // 需要手动设置
     }
     
     return false;
   }
   ```

3. **修改 `initialize()` 方法**:
   - 在初始化语音识别前，先调用 `checkAndRequestPermission()` 检查权限
   - 如果没有权限，设置 `_initializationFailed = true` 并返回 `false`

- **原因**: 
  - 确保在初始化语音识别前已获得麦克风权限
  - 提供更好的错误处理和用户反馈
  - 避免权限被拒绝时的崩溃或异常行为

### 3. 烹饪页面修改

#### ✅ **recipe_instruction_page.dart** - 修改
- **位置**: `frontend-app/lib/pages/recipes/recipe_instruction_page.dart`
- **修改内容**:

1. **添加导入**:
   ```dart
   import 'package:permission_handler/permission_handler.dart';
   ```

2. **修改 `_toggleVoiceMode()` 方法**:
   - 将方法改为 `async` 方法
   - 在开启语音模式前，先检查权限
   - 如果权限被拒绝，显示友好的错误提示
   - 如果权限被永久拒绝，提供"去设置"按钮，调用 `openAppSettings()`

   ```dart
   Future<void> _toggleVoiceMode() async {
     // 如果开启语音模式，先检查权限
     if (!_isVoiceModeActive) {
       final hasPermission = await _voiceAssistant.checkAndRequestPermission();
       if (!hasPermission) {
         // 检查是否被永久拒绝
         final status = await Permission.microphone.status;
         final isPermanentlyDenied = status.isPermanentlyDenied;
         
         // 显示错误提示
         ScaffoldMessenger.of(context).showSnackBar(
           SnackBar(
             content: Text(isPermanentlyDenied
                 ? '需要麦克风权限才能使用语音控制，请在设置中开启'
                 : '需要麦克风权限才能使用语音控制'),
             action: isPermanentlyDenied
                 ? SnackBarAction(
                     label: '去设置',
                     onPressed: () => openAppSettings(),
                   )
                 : null,
           ),
         );
         return;
       }
     }
     
     // 切换语音模式
     setState(() {
       _isVoiceModeActive = !_isVoiceModeActive;
       // ...
     });
   }
   ```

- **原因**: 
  - 在用户尝试开启语音模式时主动检查权限
  - 提供清晰的错误提示和解决方案
  - 引导用户到设置页面开启权限（如果被永久拒绝）

## 🐛 遇到的问题和解决方案

### 问题 1: Android 权限请求时机

**问题描述**:
- `speech_to_text` 包会自动请求权限，但用户体验不够好
- 如果权限被拒绝，没有明确的提示和引导

**解决方案**:
- 在开启语音模式前主动检查权限
- 如果权限未授予，先请求权限
- 如果权限被拒绝，显示友好的错误提示
- 如果权限被永久拒绝，提供跳转到设置页面的按钮

### 问题 2: 权限状态判断

**问题描述**:
- 需要区分权限的不同状态：
  - `granted`: 已授予
  - `denied`: 被拒绝（可以再次请求）
  - `permanentlyDenied`: 被永久拒绝（需要手动设置）

**解决方案**:
- 使用 `Permission.microphone.status` 检查权限状态
- 根据不同的状态采取不同的处理策略
- 对于永久拒绝的情况，引导用户到设置页面

## 📱 Android 配置验证

### Android 权限配置

**AndroidManifest.xml** 已配置:
```xml
<uses-permission android:name="android.permission.RECORD_AUDIO" />
```

**权限查询配置** (Android 11+):
```xml
<queries>
    <intent>
        <action android:name="android.speech.RecognitionService" />
    </intent>
</queries>
```

### 权限请求流程

1. **首次使用**:
   - 用户点击麦克风图标
   - 系统弹出权限请求对话框
   - 用户选择"允许"或"拒绝"

2. **权限被拒绝**:
   - 显示错误提示："需要麦克风权限才能使用语音控制"
   - 下次尝试时会再次请求权限

3. **权限被永久拒绝**:
   - 显示错误提示："需要麦克风权限才能使用语音控制，请在设置中开启"
   - 提供"去设置"按钮，点击后跳转到应用设置页面

1. **请求时机**:

2. **错误处理**:
   - 区分不同的权限状态
   - 提供友好的错误提示

3. **用户体验**:

   - 权限被永久拒绝时提供跳转到设置的快捷方式

## 🔮 后续工作建议

### 1. 优化语音命令匹配

当前 `recognizeCommand()` 方法使用 `contains()` 匹配关键词，可能导致：
- "继续" 和 "继续计时" 的匹配冲突
- 短关键词优先匹配的问题

**建议**:
- 优先匹配更长的关键词
- 使用更精确的匹配规则（如单词边界）
- 调整关键词列表，避免冲突

### 2. 错误处理优化

当前对于 `error_no_match` 错误会显示错误提示，但这是正常情况。

**建议**:
- 对 `error_no_match` 错误静默处理，自动重试
- 只对真正的错误（如权限问题、初始化失败）显示提示

### 3. 多语言支持优化

当前语音识别设置为 `zh_CN`（中文），但命令关键词支持中英文。

**建议**:
- 根据用户设置动态切换识别语言
- 或使用系统默认语言
- 优化英文命令的识别准确度

### 4. UI 调整和测试
