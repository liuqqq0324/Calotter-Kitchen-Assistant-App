## 🚨 重要提示：数据注入脚本问题

### 问题描述

虽然已经修复了 `ON CONFLICT DO UPDATE SET` 的问题，但**数据注入脚本仍然存在潜在问题**：

1. **密码哈希不匹配问题**
   - 脚本中硬编码的密码哈希 `$2a$10$N9qo8uLOickgx2ZMRZoMyeIjZAgcfl7p92ldGxad68LJZdL17lhWy` 可能不是由 `password123` 生成的
   - BCrypt 每次生成的哈希都不同（包含随机盐），但都能验证同一个密码
   - 如果脚本中的哈希值不对应 `password123`，会导致登录失败

2. **解决方案**

**方案 1: 手动录入测试用户（推荐）**

创建手动录入脚本 `manual-create-testuser.sh`，放在 `calotter-modules/calotter-inventory/test/` 目录下：

```bash
#!/bin/bash
# 手动创建 testuser（使用后端注册 API，确保密码哈希正确）

BASE_URL="http://localhost:8080"

echo "创建 testuser 用户..."

# 使用注册 API 创建用户（后端会自动生成正确的密码哈希）
RESPONSE=$(curl -s -X POST "${BASE_URL}/api/user/register" \
  -H "Content-Type: application/json" \
  -d '{
    "username": "testuser",
    "email": "testuser@example.com",
    "password": "password123"
  }')

echo "$RESPONSE" | python3 -m json.tool

# 检查是否成功
if echo "$RESPONSE" | grep -q '"code":200'; then
    echo "✓ testuser 创建成功"
    echo "✓ 密码: password123"
    echo "✓ 现在可以正常登录"
else
    echo "✗ testuser 创建失败（可能已存在）"
    echo "尝试登录验证..."
    LOGIN_RESPONSE=$(curl -s -X POST "${BASE_URL}/api/user/login" \
      -H "Content-Type: application/json" \
      -d '{"usernameOrEmail":"testuser","password":"password123"}')
    
    if echo "$LOGIN_RESPONSE" | grep -q '"code":200'; then
        echo "✓ testuser 已存在且可以登录"
    else
        echo "✗ testuser 存在但无法登录，需要手动修复"
    fi
fi
```
