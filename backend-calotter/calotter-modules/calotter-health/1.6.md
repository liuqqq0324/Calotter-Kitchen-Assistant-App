Today’s Dish Intake 功能落地（Leftover-only）
前端把 “Today’s Recipe” 改为 “Today’s Dish Intake”，弹窗展示今日条目，并支持滑条调整吃了多少百分比。
明确需求变更：不再支持 dish 类型，只支持 inventory 的 leftover。

前端改动
todays_recipes_dialog.dart：
弹窗中新增 +（选择可添加的 leftovers）。
支持 多选 + Confirm 一次性添加多道菜。
列表中显示名称字段切换为 leftoverTitle，滑条保存百分比。
homepage_api_service.dart：
新增/对齐 getDishOptions、addDishIntake、updateIntakePercentage、deleteIntake 等调用。
addDishIntake 支持 ids: [] 批量添加（兼容单个 id）。
加入 Mock 模式：--dart-define=MOCK_HOMEPAGE_API=true 用假数据验证 UI 流程（options / add / today / patch / delete）。

后端 calotter-health API 改动与对齐
新增接口：
GET /api/intake/dish/options：返回可选 leftovers 列表（来自 inventory LeftoverDish）。
POST /api/intake/dish：支持单个 id 或批量 ids[] 添加。
GET /api/intake/today?source=recipe：改为只返回 LEFTOVER 的 NutritionLog（保证可持久化/可调百分比）。
DTO 字段改名同步：
recipeId/recipeTitle → leftoverId/leftoverTitle（前后端一致）。
百分比持久化 + 同步库存：
滑条 PATCH /api/intake/{id} 更新 NutritionLog.consumedPercentage 并重算营养；
同时把百分比同步到 inventory：更新 LeftoverDish.currentQuantityGram。
批量添加时同样同步库存，并返回 addedIntakes + todayDishIntakes（兼容旧字段 intake）。
