# COMPSCI 778 – Journal Two

## 项目描述 / Project Description

Personal Sous Chef（个人厨师助手）是一个全栈智能移动应用项目，旨在通过人工智能技术帮助用户更高效地管理厨房食材、生成个性化食谱，并提供实时烹饪指导。该项目采用前后端分离的架构设计，前端使用 Flutter 框架开发跨平台移动应用，后端采用 Spring Boot 多模块架构，并集成 AI 引擎实现食材识别和智能食谱生成功能。

项目的核心价值在于解决现代家庭厨房管理中的痛点：食材浪费、食谱选择困难、烹饪过程缺乏指导等问题。通过图像识别技术自动识别食材，结合用户的饮食偏好、过敏信息和库存情况，系统能够智能推荐最适合的食谱，并在烹饪过程中提供分步骤的实时指导。

## 项目背景与研究 / Project Background and Research

### 研究背景

在当今快节奏的生活中，家庭厨房管理面临着诸多挑战。根据相关研究，家庭食物浪费是一个全球性问题，而缺乏有效的食材管理和食谱规划是导致浪费的主要原因之一。传统的食材管理方式依赖人工记录，效率低下且容易出错。同时，用户在选择食谱时往往面临信息过载的问题，难以找到既符合个人偏好又能够充分利用现有食材的食谱。

### 技术研究现状

在食材识别领域，计算机视觉技术已经取得了显著进展。深度学习模型，特别是 YOLO（You Only Look Once）系列模型，在目标检测任务中表现出色，能够实现实时、高精度的物体识别。本项目计划使用 YOLO 模型进行食材识别，通过训练自定义数据集来识别常见的厨房食材。

在食谱生成方面，生成式 AI 技术（如大型语言模型）已经能够根据给定的食材和用户偏好生成高质量的食谱内容。通过构建合适的提示词（prompt）和上下文信息，AI 可以生成个性化的、详细的烹饪步骤。

在系统架构方面，RESTful API 设计已经成为现代 Web 应用的标准。REST（Representational State Transfer）架构风格提供了统一接口、无状态操作、可缓存响应等特性，使得系统具有良好的可扩展性和可维护性（Fielding, 2000）。本项目采用 RESTful API 设计，确保前后端之间的数据交互清晰、规范。

### 相关技术文献

- **REST 架构风格**：Fielding (2000) 在其博士论文中提出了 REST 架构的六个约束条件，包括统一接口、无状态操作、可缓存响应等，这些原则指导了本项目的 API 设计。

- **深度学习在图像识别中的应用**：YOLO 系列模型在实时目标检测任务中表现出色，适合移动端和边缘设备的部署需求。

- **微服务架构**：本项目采用 Spring Boot 多模块架构，借鉴了微服务的设计思想，将不同业务功能模块化，提高了系统的可维护性和可扩展性。

## 项目目标与目标 / Project Goals and Objectives

### 核心目标

1. **开发一个功能完整的智能厨房管理应用**
   - 实现用户账户管理和个性化设置
   - 提供直观、易用的用户界面
   - 支持跨平台部署（iOS 和 Android）

2. **实现基于 AI 的食材识别功能**
   - 通过图像/视频扫描自动识别食材
   - 支持用户手动修正识别结果
   - 提供高精度的食材识别准确率

3. **提供智能食谱生成和推荐**
   - 基于用户库存自动生成可用食谱
   - 考虑用户饮食偏好、过敏信息和健康目标
   - 提供食谱搜索和筛选功能

4. **实现烹饪过程指导功能**
   - 提供分步骤的烹饪指南
   - 支持免提模式（语音导航）
   - 内置计时器功能

### 功能模块目标

#### 1. 用户账户与个性化模块 [FR1.0.0]
- 实现用户注册、登录功能（支持邮箱和第三方登录）
- 用户资料管理（年龄、性别、身高、体重）
- 个性化设置（饮食偏好、禁忌、过敏信息）

#### 2. 库存管理模块 [FR2.0.0]
- 图像/视频扫描识别食材（核心 AI 功能）
- 手动添加、编辑、删除食材
- 食材数量与过期日期管理
- 调料和炊具管理

#### 3. 食谱生成与管理模块 [FR3.0.0]
- 基于库存和用户偏好生成个性化食谱（核心 AI 功能）
- 食谱搜索与筛选
- 显示所需食材及消耗量
- 生成购物清单（缺失食材）

#### 4. 烹饪助手模块 [FR4.0.0]
- 分步骤烹饪指南
- 免提模式（语音导航）
- 内置计时器
- 实时反馈（可选功能）

### 技术挑战与解决方案

**预期挑战**

1. **AI 模型准确率**
   - **挑战**：食材识别准确率可能不够高，特别是对于相似食材的区分
   - **解决方案**：增加训练数据量，使用数据增强技术，实现用户反馈机制来持续改进模型

2. **前后端数据同步**
   - **挑战**：确保前后端数据一致性，处理网络异常情况
   - **解决方案**：实现离线数据缓存，使用乐观更新策略，添加数据同步机制

3. **AI 服务性能**
   - **挑战**：图像识别和食谱生成可能需要较长的响应时间
   - **解决方案**：实现异步处理，使用消息队列，优化模型推理速度，添加缓存机制

4. **用户体验优化**
   - **挑战**：确保应用在各种设备上都能流畅运行
   - **解决方案**：进行多设备测试，优化图像加载和渲染性能，实现响应式设计

## 总结 / Summary

Personal Sous Chef 项目目前已经完成了前端原型的开发和后端核心架构的搭建。前端使用 Flutter 实现了完整的用户界面和交互流程，可以作为演示版本展示项目的设计理念和用户体验。后端采用 Spring Boot 多模块架构，实现了用户管理和库存管理等核心功能，API 接口设计遵循 RESTful 规范，具有良好的可扩展性。

项目的下一步重点是前后端集成和 AI 功能的开发。通过集成 YOLO 模型实现食材识别，以及集成生成式 AI 实现智能食谱生成，项目将实现其核心价值。同时，通过持续的用户测试和反馈，不断优化用户体验和系统性能。

这个项目不仅是一个技术实践，更是对如何利用现代技术解决实际生活问题的探索。通过结合移动应用开发、后端系统设计、人工智能技术等多个领域，我们希望能够为用户提供一个真正有用的智能厨房管理工具。
