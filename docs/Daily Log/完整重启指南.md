## 步骤 4: 初始化数据库

### 4.1 执行主初始化脚本（创建数据库和基础 schema）

**重要说明：** `db_init.sql` 文件包含：
- 数据库的删除和创建
- 4 个 schema 的创建（cms, ims, rms, ums）- **但缺少 hp schema**
- 这些 schema 下所有表的创建（CMS, IMS, RMS, UMS 的表）

由于 `db_init.sql` 包含 `\c` 命令（切换数据库），无法直接通过 `docker exec` 执行整个文件。有两种方式：

#### 方式 1：分步执行（推荐）

```bash
# 进入数据库目录
cd /Users/chase/Documents/Internship/Projects/chef_git/A-team-PersonalSousChef/calotter/db

# 步骤 1: 删除并创建数据库
docker exec postgres psql -U postgres -c "SELECT pg_terminate_backend(pid) FROM pg_stat_activity WHERE datname = 'sous_chef' AND pid <> pg_backend_pid();" 2>/dev/null || true
docker exec postgres psql -U postgres -c "DROP DATABASE IF EXISTS sous_chef;"
docker exec postgres psql -U postgres -c "CREATE DATABASE sous_chef;"

# 步骤 2: 创建所有 schema（包括 HP，因为 db_init.sql 中缺少）
docker exec postgres psql -U postgres -d sous_chef -c "CREATE SCHEMA IF NOT EXISTS sous_chef_cms;"
docker exec postgres psql -U postgres -d sous_chef -c "CREATE SCHEMA IF NOT EXISTS sous_chef_ims;"
docker exec postgres psql -U postgres -d sous_chef -c "CREATE SCHEMA IF NOT EXISTS sous_chef_rms;"
docker exec postgres psql -U postgres -d sous_chef -c "CREATE SCHEMA IF NOT EXISTS sous_chef_ums;"
docker exec postgres psql -U postgres -d sous_chef -c "CREATE SCHEMA IF NOT EXISTS sous_chef_hp;"

# 步骤 3: 执行 db_init.sql 中的表创建部分
# 由于 db_init.sql 包含 \c 命令，我们需要手动提取表创建部分
# 或者直接执行各个 schema 目录下的 SQL 文件（见步骤 4.3）
```

#### 方式 2：执行 db_init.sql 的表创建部分（如果文件可以修改）

如果你想执行 `db_init.sql` 中的表创建，需要先修改文件，删除 `\c` 命令，然后：
```bash
# 修改后的 db_init.sql（删除数据库创建部分，只保留表创建）
docker exec -i postgres psql -U postgres -d sous_chef < db_init_modified.sql
```

**推荐使用方式 1**，然后继续执行步骤 4.3 中的各个 schema 的 SQL 文件。

### 4.2 创建 HP Schema（已在步骤 4.1 中创建）

HP Schema 已在步骤 4.1 中创建，如果之前没有创建，可以执行：

## 🔧 常见问题排查

```bash

# 重启容器
```

### 问题 2: Schema 不存在错误

如果看到 `schema "sous_chef_hp" does not exist` 错误：
