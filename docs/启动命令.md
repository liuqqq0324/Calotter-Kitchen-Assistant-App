# 完整项目启动命令

## 前置要求

- Java 17+
- Maven 3.6+
- PostgreSQL 15+ (或使用 Docker)
- Flutter SDK (用于前端)

---

## 1. 启动数据库（PostgreSQL）

### 方式1：使用 Docker Compose（推荐）

```bash
# 进入 calotter 目录
cd calotter

# 启动 PostgreSQL 和 Redis
docker-compose up -d

# 检查容器状态
docker ps
```

**数据库配置：**
- 主机：`127.0.0.1`
- 端口：`5432`
- 用户名：`postgres`
- 密码：`123`
- 数据库：`sous_chef`（需要手动创建）

### 方式2：使用本地 PostgreSQL

```bash
# 确保 PostgreSQL 服务运行
# macOS
brew services start postgresql@15

# Linux
sudo systemctl start postgresql
```

## 5. 一键启动脚本（推荐）

创建启动脚本 `start-all.sh`：

```bash
#!/bin/bash

# 项目根目录
PROJECT_ROOT="/Users/chase/Documents/Internship/Projects/chef_git/A-team-PersonalSousChef"
cd "$PROJECT_ROOT"

echo "🚀 启动完整项目..."

# 1. 启动数据库
echo "📦 启动 PostgreSQL 和 Redis..."
cd calotter
docker-compose up -d
sleep 5

# 2. 启动 calotter-user 服务
echo "👤 启动 calotter-user 服务（端口 10000）..."
cd calotter-user
mvn clean spring-boot:run > ../logs/user.log 2>&1 &
USER_PID=$!
echo "User service PID: $USER_PID"
sleep 10

# 3. 启动 calotter-homepage 服务
echo "🏠 启动 calotter-homepage 服务（端口 10001）..."
cd ../calotter-homepage
mvn clean spring-boot:run > ../logs/homepage.log 2>&1 &
HOMEPAGE_PID=$!
echo "Homepage service PID: $HOMEPAGE_PID"
sleep 10

# 4. 启动前端（可选）
echo "📱 启动前端应用..."
cd ../../frontend-app
flutter run > ../logs/frontend.log 2>&1 &
FRONTEND_PID=$!
echo "Frontend PID: $FRONTEND_PID"

echo "✅ 所有服务已启动！"
echo "User Service: http://localhost:10000"
echo "Homepage Service: http://localhost:10001"
echo ""
echo "查看日志："
echo "  tail -f calotter/logs/user.log"
echo "  tail -f calotter/logs/homepage.log"
echo ""
echo "停止服务："
echo "  kill $USER_PID $HOMEPAGE_PID $FRONTEND_PID"
```

**使用脚本：**

```bash
# 创建日志目录
mkdir -p calotter/logs

# 添加执行权限
chmod +x start-all.sh

# 运行脚本
./start-all.sh
```

## 6. 停止服务

### 停止 Docker 容器

```bash
cd calotter
docker-compose down
```

### 停止后端服务

```bash
# 查找 Java 进程
ps aux | grep java

# 停止特定服务
pkill -f "calotter-user"
pkill -f "calotter-homepage"

# 或者使用 PID
kill <PID>
```

### 停止前端

```bash
# 在运行 Flutter 的终端按 Ctrl+C
# 或者
pkill -f flutter
```

## 8. 常见问题

### 问题1：端口被占用

```bash
# 查找占用端口的进程
lsof -i :10000
lsof -i :10001

# 杀死进程
kill -9 <PID>
```

### 问题2：数据库连接失败

- 检查 PostgreSQL 是否运行：`docker ps` 或 `brew services list`
- 检查数据库配置：`calotter/calotter-*/src/main/resources/application.yml`
- 检查数据库是否创建：`psql -U postgres -l | grep sous_chef`

### 问题3：编译错误

```bash
# 清理并重新编译
mvn clean install

# 如果仍有问题，检查 Lombok 配置
# 确保 pom.xml 中有 Lombok 注解处理器配置
```

### 问题4：前端无法连接后端

- 检查 `frontend-app/lib/config/api_config.dart` 中的服务器 IP 和端口
- 确保后端服务已启动
- 检查防火墙设置

## 9. 开发环境快速启动（推荐）

### 使用 IDE（IntelliJ IDEA / VS Code）

1. **启动数据库**
   ```bash
   cd calotter && docker-compose up -d
   ```

2. **在 IDE 中运行服务**
   - 打开 `calotter/calotter-user/src/main/java/com/calotter/user/CalotterUserApplication.java`
   - 右键 → Run
   - 打开 `calotter/calotter-homepage/src/main/java/com/calotter/homepage/CalotterHomepageApplication.java`
   - 右键 → Run

3. **运行前端**
   ```bash
   cd frontend-app && flutter run
   ```

## 10. 生产环境部署

### 打包后端服务

```bash
# 打包所有服务
cd calotter
mvn clean package

# 生成的 JAR 文件在：
# calotter/calotter-user/target/calotter-user-0.0.1-SNAPSHOT.jar
# calotter/calotter-homepage/target/calotter-homepage-0.0.1-SNAPSHOT.jar
```

### 运行生产版本

```bash
# 使用 nohup 后台运行
nohup java -jar calotter-user/target/calotter-user-0.0.1-SNAPSHOT.jar > logs/user.log 2>&1 &
nohup java -jar calotter-homepage/target/calotter-homepage-0.0.1-SNAPSHOT.jar > logs/homepage.log 2>&1 &
```
