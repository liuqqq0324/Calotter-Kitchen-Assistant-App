# Personal Sous Chef 环境设置指南

本文档提供项目的完整环境设置和运行操作指南。

## 项目结构

- **frontend-app/**: Flutter 移动应用前端
- **backend-api/**: 后端 API 服务
- **ai-engine/**: AI 引擎服务
- **database/**: 数据库初始化脚本

## 前置要求

### 1. 安装 Flutter SDK

**macOS 安装步骤：**

1. 下载 Flutter SDK：
   - 访问 https://flutter.dev/docs/get-started/install/macos
   - 下载最新的稳定版本

2. 解压并设置环境变量：
   ```bash
   # 将 Flutter 解压到您想要的位置，例如：
   cd ~/development
   unzip ~/Downloads/flutter_macos_*.zip
   
   # 添加到 PATH（在 ~/.zshrc 或 ~/.bash_profile 中）：
   export PATH="$PATH:$HOME/development/flutter/bin"
   ```

3. 运行 Flutter 医生检查：
   ```bash
   flutter doctor
   ```

4. 安装 Xcode（iOS 开发需要）：
   ```bash
   # 通过 App Store 安装 Xcode
   # 然后运行：
   sudo xcode-select --switch /Applications/Xcode.app/Contents/Developer
   sudo xcodebuild -runFirstLaunch
   ```

5. 安装 CocoaPods（iOS 依赖管理）：
   ```bash
   sudo gem install cocoapods
   ```

### 2. 安装 Docker 和 Docker Compose

**macOS 安装步骤：**

1. 下载 Docker Desktop for Mac：
   - 访问 https://www.docker.com/products/docker-desktop
   - 下载并安装 Docker Desktop

2. 验证安装：
   ```bash
   docker --version
   docker-compose --version
   ```

### 3. 安装 Python（AI 引擎需要）

**macOS 安装步骤：**

1. 使用 Homebrew 安装 Python：
   ```bash
   brew install python@3.11
   ```

2. 或者从官网下载：
   - 访问 https://www.python.org/downloads/
   - 下载并安装 Python 3.11 或更高版本

3. 验证安装：
   ```bash
   python3 --version
   pip3 --version
   ```

## 环境设置步骤

### 步骤 1: 克隆项目（如果还没有）

```bash
cd /Users/chase/Documents/Internship/Projects/chef_git
git clone <repository-url> A-team-PersonalSousChef

```

### 步骤 2: 设置 Flutter 前端

1. 进入前端目录：
   ```bash
   cd frontend-app
   ```

2. 获取 Flutter 依赖：
   ```bash
   flutter pub get
   ```

3. 检查 Flutter 环境：
   ```bash
   flutter doctor
   ```

4. 查看可用设备：
   ```bash
   flutter devices
   ```

### 步骤 3: 设置后端服务（Docker）

1. 返回项目根目录：
   ```bash
   cd ..
   ```

2. 查看 docker-compose.yml 配置（确认服务配置）

3. 启动所有服务：
   ```bash
   docker-compose up -d
   ```

4. 查看运行状态：
   ```bash
   docker-compose ps
   ```

5. 查看日志：
   ```bash
   docker-compose logs -f
   ```

### 步骤 4: 设置 AI 引擎（如果需要本地开发）

1. 进入 AI 引擎目录：
   ```bash
   cd ai-engine
   ```

2. 创建虚拟环境（推荐）：
   ```bash
   python3 -m venv venv
   source venv/bin/activate  # macOS/Linux
   ```

3. 安装依赖：
   ```bash
   pip install -r requirements.txt
   ```

## 运行项目

### 运行 Flutter 应用

1. 进入前端目录：
   ```bash
   cd frontend-app
   ```

2. 运行应用（选择一种方式）：
   ```bash
   # 在连接的设备/模拟器上运行
   flutter run
   
   # 指定设备运行
   flutter run -d <device-id>
   
   # 在 Chrome 浏览器中运行（Web）
   flutter run -d chrome
   
   # 在 iOS 模拟器中运行
   flutter run -d ios
   
   # 在 Android 模拟器中运行
   flutter run -d android
   ```

3. 热重载：
   - 在运行的应用中按 `r` 进行热重载
   - 按 `R` 进行热重启
   - 按 `q` 退出

### 运行后端服务

**使用 Docker Compose（推荐）：**

```bash
# 在项目根目录
docker-compose up -d          # 后台运行
docker-compose up             # 前台运行（查看日志）
docker-compose down           # 停止服务
docker-compose restart        # 重启服务
```

**单独运行服务：**

```bash
# 只启动数据库
docker-compose up -d database

# 只启动后端 API
docker-compose up -d backend-api

# 只启动 AI 引擎
docker-compose up -d ai-engine
```

### 运行 AI 引擎（本地开发模式）

```bash
cd ai-engine
source venv/bin/activate  # 如果使用虚拟环境
python app.py  # 或相应的启动脚本
```

## 常见问题排查

### Flutter 问题

1. **Flutter doctor 显示问题：**
   - 按照提示安装缺失的组件
   - 运行 `flutter doctor -v` 查看详细信息

2. **依赖安装失败：**
   ```bash
   flutter clean
   flutter pub get
   ```

3. **iOS 构建问题：**
   ```bash
   cd ios
   pod install
   cd ..
   ```

### Docker 问题

1. **端口冲突：**
   - 检查 docker-compose.yml 中的端口配置
   - 确保端口未被其他服务占用

2. **容器无法启动：**
   ```bash
   docker-compose logs <service-name>
   # 查看具体错误信息
   ```

3. **镜像构建失败：**
   ```bash
   docker-compose build --no-cache
   ```

### 数据库问题

1. **数据库连接失败：**
   - 检查 docker-compose.yml 中的数据库配置
   - 确认数据库容器正在运行
   - 检查连接字符串和凭据
