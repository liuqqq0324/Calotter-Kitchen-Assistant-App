## 3. ☕ 后端开发指南 (Backend Devs)
后端部署流程已完全自动化。你只需要专注于写代码。

🚀 如何更新服务器代码？
在本地修改代码、修复 Bug。

提交代码：git commit -m "feat: 新增了xxx功能"

推送到仓库: git push

就在你 Push 的那一瞬间，自动化流程启动：

GitHub 会自动打包你的代码。

自动上传到 AWS。

自动重启服务器（智能清理旧进程，启动新 Jar 包）。

等待约 2-3 分钟，AWS 上的接口就会变成你最新的版本。

⚠️ 注意事项 (冲突管理)
目前配置为 任意分支 Push 都会触发部署。

请注意： 如果多人同时 Push，后提交的人会覆盖先提交的人的版本。

建议： 大家在群里同步一下，或者在合并到 main 分支时再进行重要测试。

5. 🚑 故障排查 (Troubleshooting)
如果发现接口挂了（500 Error 或连不上）：

第一步：检查部署状态
去 GitHub 仓库点击顶部菜单的 "Actions" 标签页。

🟡 Yellow: 正在部署中，请等待 1-2 分钟。

✅ Green: 部署成功。

❌ Red: 编译或启动失败，请点进去看报错日志。

第二步：查看服务器实时日志（仅限管理员）
如果 GitHub 显示成功但网页打不开，联系管理员 (@Emma) 登录服务器查看日志：

Bash

# 登录服务器
ssh -i Kitchen-key.pem ubuntu@16.176.170.214

# 查看实时日志
tail -f /opt/calotter/app.log
